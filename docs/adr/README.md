# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) documenting significant technical decisions made during the development of ADS for Overleaf.

## Index

| ADR | Title | Status |
|-----|-------|--------|
| [ADR-001](001-manifest-v3.md) | Use Chrome Manifest V3 | Accepted |
| [ADR-002](002-no-build-tooling.md) | No Build Tooling | Accepted |
| [ADR-003](003-local-storage.md) | Local Storage vs Sync Storage | Accepted |
| [ADR-004](004-cache-strategy.md) | TTL-Based Caching Strategy | Accepted |
| [ADR-005](005-content-script-architecture.md) | Content Script Architecture | Accepted |

## ADR Format

Each ADR follows this structure:
- **Status**: Proposed, Accepted, Deprecated, Superseded
- **Context**: Why this decision was needed
- **Decision**: What was decided
- **Consequences**: Positive and negative outcomes
